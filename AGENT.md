# AGENT.md

## 運用方針

- `Docs` ディレクトリ配下に各種ドキュメントを作成していく。
- `powershell版` ディレクトリには、Windows 環境で GUI 利用を前提としたスクリプトを配置する。
- `PHP版` ディレクトリを作成し、ブラウザから利用できるバージョンを開発していく。
- このプロジェクトは仕様駆動開発で進める。
-- 仕様書が未作成の場合は必ず作成して、人に確認してから実装を開始すること。

## ツール概要

- 本ツールは、週次スケジュールを ESC/POS 対応プリンタへ印刷するためのツールである。
- 現在は Windows 向け PowerShell GUI 版を提供し、将来的にブラウザ利用向け PHP 版も提供する。
- ユーザーは開始日（週）を選択し、プレビューを確認・必要に応じて編集して印刷できる。
- 印刷先は IP / Port / Timeout を画面から指定し、TCP 経由で印字データを送信する。
- 週末強調などの印字指定は `!!...!!` マーカーで表現し、印刷時に反転印字へ変換する。
- 印刷時の内容は週単位バックアップファイルとして保存し、同じ週を再度開いた際に再利用できる。

## powershell版の実装・運用ルール

- `powershell版` のスクリプトは `#requires -version 5.1` を満たすことを前提とする。
- GUI は `System.Windows.Forms` / `System.Drawing` を用いた Windows 向け実装とする。
- 印刷は ESC/POS を前提とし、ネットワークプリンタへ TCP（既定ポート `9100`）で送信する。
- 印字データの文字コードは Shift_JIS を前提に扱う。
- 週次の編集内容は `powershell版` 配下に `m5-weekly-refill-YYYY-MM-DD.bak.txt` 形式（UTF-8）で保存する。
- 週テンプレートは月曜始まりで生成し、`!!...!!` マーカーは反転印字指定として扱う。

## PHP版の開発方針

- `PHP版` ディレクトリで、ブラウザから利用できる実装を段階的に追加する。
- 仕様・画面・入出力の定義は仕様駆動で `Docs` 配下に先に文書化してから実装する。
- 印刷要件（週テンプレート、マーカー記法、通信条件）は PowerShell 版との整合性を維持する。
